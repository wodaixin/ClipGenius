# 历史记录管理

应用右侧的历史面板展示所有已捕获的剪贴板条目。排序规则：置顶项目优先，然后按时间戳（最新的在前）。

## 搜索

历史面板顶部的搜索栏使用 [Fuse.js](https://fusejs.io/) 对所有条目进行模糊搜索。

搜索目标：
- `suggestedName`（权重：0.4）
- `summary`（权重：0.3）
- `content`（权重：0.3）

阈值：0.4（相当宽松的模糊匹配）。输入时实时更新结果。

清空搜索：清空搜索字段或按 Escape 键。

## 预览

点击任意卡片打开 `PastePreview` 弹窗，渲染完整内容：
- 图片/视频：通过 `<img>` / `<video>` 标签渲染
- 代码：通过 `react-syntax-highlighter` 进行语法高亮
- Markdown：通过 `react-markdown` 和 `rehype-highlight` 渲染
- 链接：显示为可点击链接
- 纯文本：格式化为文本显示

## 置顶 / 取消置顶

点击任意卡片上的图钉图标将其置顶。置顶项目：
- 始终显示在列表顶部
- 不会被「清除未置顶」操作删除

置顶操作会更新条目的 `isPinned` 字段并触发同步写入。

## 编辑名称和摘要

双击任意卡片上的名称或摘要，或在编辑字段获得焦点后点击保存按钮来保存更改。`usePasteStore` 中的 `editingItemId` 状态跟踪正在编辑的条目。`tabSync` 中的 `editingIds` Set 防止一个标签页覆盖另一个标签页的编辑。

## 删除

点击任意卡片上的删除图标进行软删除。条目被标记 `isDeleted: true` 并设置 `deletedAt` 时间戳，从 UI 立即移除，并在已登录时同步到 Firestore。

## 清除未置顶

「清除未置顶」操作（可在历史面板顶部或右键菜单中找到）对所有未置顶项目进行批量软删除。置顶项目会被保留。该操作是乐观的：条目立即从 UI 移除，Firestore 写入异步进行。

## 内容预览截断

超过 2000 个字符的文本内容在卡片预览中被截断（存储在 React 状态中），以避免渲染过大的 DOM 节点。完整内容保留在 IndexedDB 中。
