# 多模态 AI 聊天

ClipGenius 内置了功能完整的 AI 聊天助手，可以与剪贴板内容进行多模态对话。

## 聊天会话标识

每个聊天会话通过唯一 ID 标识：

- **附加剪贴内容时**：聊天 ID = 附加的 PasteItem ID
- **无附加内容时**：聊天 ID = `"default"`

这确保了每个剪贴项目都有独立的对话历史。

## 上下文注入

与剪贴内容对话时，系统会自动将内容作为上下文注入：

### 图片/视频上下文

```
[上下文：附加了图片]
用户问题：这是什么？
```

### 文本上下文

```
上下文："这是剪贴的文本内容..."

用户问题：这段话的主要观点是什么？
```

## 流式响应架构

聊天采用流式架构：

1. 用户发送消息
2. AI 逐步返回响应内容
3. 支持 `thinking` 流，展示推理过程
4. 支持 Web 搜索结果集成

## 消息历史限制

为控制 token 使用量，消息历史有上限：

- 保留最近的消息进行上下文
- 超出限制的消息会被截断

## 双重持久化

聊天消息同时保存到两个位置：

1. **IndexedDB**：本地持久化，访客可用
2. **Firestore**：云端同步，登录用户可用

```typescript
// Firestore 路径结构
/users/{userId}/chats/{chatId}/messages/{messageId}
```

## Firestore 合并策略

云端数据采用合并写入：

```typescript
await setDoc(docRef, messageData, { merge: true });
```

这确保：
- 新字段被添加
- 现有字段被更新
- 未提及的字段保持不变

## 错误处理

连接失败时：
- 显示错误提示
- 允许用户重试
- 消息不会丢失（本地已保存）

## 思考过程展示

当 AI 模型支持推理功能时，思考过程会单独展示：

```
思考中...（thinking 内容）
↓（思考结束）
实际回复内容
```
